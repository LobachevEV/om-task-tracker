# Plan fixtures

All Gantt / Plan stories import from `FeatureFixtures.ts`. **Never** use `new Date()` or
`Date.now()` in stories — time is deterministic here.

`FIXTURE_TODAY = "{FIXTURE_TODAY}"` is the only reference point. Any "overdue" or
"shipped" framing derives from this literal, not the real clock. If a story needs a
variant not covered below, add it to the fixture file and import it — do **not**
inline ids or dates at the story callsite.

## `FeatureDetail` fixture

`MINI_TEAM_FEATURE_DETAIL` hangs off `MINI_TEAM_FEATURE` and adds 5 `AttachedTask`
rows distributed across `be`, `fe`, `qa`:

| Task `id` | `jiraId` | `state` | `userId` |
| --- | --- | --- | --- |
{MINI_TEAM_FEATURE_DETAIL.tasks.map((t) => (
  `| ${t.id} | \`${t.jiraId}\` | \`${t.state}\` | ${t.userId} |`
)).join('\n')}

`lead` is `be` (`BackendDeveloper`). `miniTeam` is `[be, fe, qa]`.

## State → token mapping

See `features/gantt/stateConfig.ts` for the authoritative `FeatureState → --state-*`
table. Tokens themselves are documented under
[Foundations / Colors](./?path=/docs/foundations-colors--docs) and declared in
`src/styles/tokens.css`. No new tokens are introduced by the Plan feature — every
bar segment resolves to an existing `--state-*` value.
